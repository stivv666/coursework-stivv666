Тема варіанту
Розробка консольного додатку «Фінансовий менеджер» для обліку особистих фінансів.


Опис проєкту
Проєкт являє собою консольну програму на мові C#, створену для контролю за домашньою бухгалтерією. Додаток дозволяє користувачеві фіксувати свої доходи та витрати, розбивати їх по категоріям та відслідковувати загальний поточний баланс. Ключовою особливістю системи є можливість встановлювати бюджетні ліміти на конкретні категорії витрат: якщо витрати перевищують ліміт, система генерує попередження.


Основні функції
Перегляд історії: Виведення всіх транзакцій у зручному табличному форматі з відображенням дати, категорії, суми та опису.

Керування категоріями: Додавання нових категорій з можливістю (за бажанням) вказати ліміт бюджету.

Фіксація грошових рухів: Додавання транзакцій типу "Дохід" та "Витрата".

Математика балансу: Автоматичний підрахунок поточного балансу як різниці між усіма доходами та витратами.

Керування даними (CRUD): Можливість безпечного видалення будь-якої транзакції зі списку.

Валідація: Блокування збереження транзакцій чи категорій, якщо вони містять некоректні дані (наприклад, пусті рядки або від'ємні суми).


Архітектура проєкту
Проєкт побудовано за принципами багатошарової архітектури (N-Tier Architecture), що забезпечує чітке розділення відповідальності:

Domain: Містить базові моделі та сутності (Transaction, Category).

DataAccess: Відповідає за роботу з файловою системою (збереження та читання JSON-файлів) через загальний інтерфейс IRepository.

Services: Містить шар бізнес-логіки (FinanceService), який обробляє дані та приймає рішення про перевищення лімітів.

Validation: Окремий модуль для перевірки правильності введених даних перед їх передачею в базу.

UI: Інтерфейс користувача (ConsoleManager), який відповідає виключно за взаємодію з консоллю (малювання таблиць, зчитування кнопок).


Використання технологій і конструкцій
Мова та платформа: C# та .NET 10.

Об'єктно-орієнтоване програмування (ООП): Широке використання класів та інтерфейсів (наприклад, IFinanceService), що дозволяє легко тестувати код.

Узагальнення (Generics): Використання патерну Generic Repository (IRepository<T>) для універсальної роботи як з транзакціями, так і з категоріями.

LINQ (Language Integrated Query): Використовується в сервісі для швидкої фільтрації колекцій та підрахунку сум (Where, Sum, FirstOrDefault).

Події та делегати (Events & Delegates): Використання події event Action<string> OnBudgetExceeded для реалізації механізму сповіщень про перевищення бюджету.

Рефлексія (Reflection API): Створення кастомних атрибутів ([PositiveValue], [RequiredString]) та виклик Validator.Validate для динамічної перевірки властивостей об'єктів під час виконання програми.

Модульне тестування (Unit Testing): Використання фреймворку NUnit для автоматичної перевірки бізнес-логіки системи (написано 10 тестів, що покривають позитивні, негативні та граничні сценарії).


Запуск проєкту
Запуск основної програми: Відкрити термінал у кореневій папці проєкту (там, де файл .csproj або .sln) і ввести команду dotnet run.

Запуск тестів: Для перевірки працездатності логіки перейти в терміналі до папки з тестовим проєктом (FinanceManager.Tests) та виконати команду dotnet test.
